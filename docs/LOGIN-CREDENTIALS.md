# بيانات تسجيل الدخول (للتطوير والاختبار)

## تسجيل الدخول كمسؤول (لوحة التحكم)

| الحقل | القيمة |
|--------|--------|
| **البريد الإلكتروني** | `admin@example.com` |
| **كلمة المرور** | `Admin123!` |

- الاستخدام: صفحة تسجيل الدخول في **لوحة التحكم (Dashboard)** على المتصفح.
- الرابط: مثلاً `http://localhost:3001` أو حسب إعداد المشروع.

---

## تسجيل الدخول كمريض (تطبيق المريض)

| الحقل | القيمة |
|--------|--------|
| **رقم الجوال** | `+966501234567` |
| **كلمة المرور** | `Patient123!` |

- الاستخدام: تطبيق المريض (React Native / Expo) — حقل "رقم الجوال" + "كلمة المرور".
- بعد تشغيل الـ API وتشغيل `npm run db:seed` في مجلد `backend` سيتم إنشاء المستخدمين الافتراضيين (أدمن + مريض) إن لم يكونوا موجودين.

---

## إنشاء البيانات الافتراضية

من مجلد الـ backend:

```bash
cd backend
npm run db:seed
```

يُنشئ السكربت:
- مستخدم **مسؤول** (admin) بالبريد وكلمة المرور أعلاه.
- مستخدم **مريض** (patient) بالجوال وكلمة المرور أعلاه مع سجل مريض مرتبط.
