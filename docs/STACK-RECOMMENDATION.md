# توصية التقنيات (Stack Recommendation)

لمركز العلاج الفيزيائي — سرعة تنفيذ، استقرار عالٍ، وأقوى نماذج مناسبة.

---

## Backend
- **Node.js (NestJS)** أو **Python (FastAPI)**
- **لماذا؟** الأفضل في التعامل مع التنبيهات اللحظية (Real-time) ورفع الملفات الكبيرة (مثل الأشعة)، وأداء جيد مع I/O.

## Frontend (Web — لوحة التحكم)
- **React.js** مع **Tailwind CSS**
- **لماذا؟** سرعة تصفح وتطوير لوحة تحكم للأطباء والاستقبال، ومجتمع كبير ومكونات جاهزة.

## Mobile App
- **Flutter**
- **لماذا؟** تطبيق واحد لأندرويد وآيفون للسائقين والمرضى والأطباء، أداء قريب من الأصلي، وواجهة موحدة.

## Database
- **PostgreSQL** — قوة علاقات البيانات الطبية، JSON حيث يلزم، وموثوقية عالية.
- **Redis** — للسرعة في تخزين المواعيد المؤقتة، الجلسات، والكاش (مثلاً أوقات متاحة، قوائم).

## Storage (الملفات والأشعة)
- **AWS S3** أو **Google Cloud Storage**
- تخزين صور الأشعة والملفات بشكل آمن ومشفر (encryption at rest)، مع صلاحيات وصول دقيقة.

## Phase 1 التقنية المختارة
- **Backend:** NestJS (Node.js) + PostgreSQL + Redis (للمواعيد/الجلسات لاحقاً).
- **Auth:** JWT مع تحديد الصلاحيات (Roles: Admin, Doctor, Patient, Driver).
- البداية بـ Schema للمستخدمين والمرضى ثم المواعيد.
